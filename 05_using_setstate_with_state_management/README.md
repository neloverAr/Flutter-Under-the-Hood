# using_setstate_with_state_management

## Do You Still Need setState with State Management?

هذا المشروع يوضح كيف يمكن لـ `setState` و `GetX` العمل معاً في تطبيق واحد بشكل متكامل، حيث يتم توزيع المهام بناءً على نوع الحالة (State Type).

### فلسفة العمل وتوزيع الأدوار:

1.  **إدارة الحالة المشتركة (Global State) باستخدام GetX:**
    *   تم استخدام **GetX** لإدارة العداد (Shared Counter) الموجود في `Screen1` و `Screen2`.
    *   يتم حقن الكنترولر في بداية التطبيق، مما يسمح لأي شاشة بالوصول لنفس البيانات وتحديثها فوراً.
    *   هذا هو المكان الصحيح لاستخدام مكتبات إدارة الحالة؛ عندما تكون البيانات مشتركة وتتعلق بمنطق العمل (Business Logic).

2.  **إدارة الحالة المحلية (Local UI State) باستخدام setState:**
    *   في **Screen1**: يتم استخدام `setState` لتغيير لون مربع التمييز (Highlighting) عند الضغط عليه. هذه حالة بصرية بحتة لا تحتاج للمشاركة مع شاشات أخرى.
    *   في **Screen2**: يتم استخدام `setState` لإظهار أو إخفاء نص تفصيلي (Toggle Details).
    *   استخدام `setState` هنا يقلل من تعقيد الكود ويحافظ على بساطة الـ Controllers، حيث أن هذه الحالات "لحظية" (Ephemeral) وتنتهي بمجرد مغادرة الشاشة.

3.  **التنقل (Navigation):**
    *   يتم استخدام `setState` في `MainNavigationScreen` لإدارة التبويب الحالي في الـ `BottomNavigationBar` والتبديل بين الشاشات باستخدام `IndexedStack` للحفاظ على حالة كل شاشة.

**الخلاصة:** أفضل ممارسة هي عدم استبدال `setState` بالكامل، بل استخدامها للواجهات المحلية، وترك المهام الثقيلة والمشتركة لمكتبات مثل `GetX`.
