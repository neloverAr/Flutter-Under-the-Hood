# Keys vs No Keys in Lists

This project demonstrates the fundamental difference in how Flutter handles Widgets and their associated State when dealing with dynamic lists, specifically when items are removed.

### 🔍 About the Project
The application features two identical screens for comparison:
1.  **No Keys:** Demonstrates how Flutter might reuse the State of a widget for a different item if they share the same type, leading to UI inconsistencies (like incorrect colors) when elements are removed.
2.  **With Keys:** Shows how using `UniqueKey` ensures that Flutter correctly maps each widget to its corresponding State, maintaining data and visual consistency even after item removal.

### 🛠 Technologies Used
*   **Flutter StatefulWidgets:** To observe how State persists in memory.
*   **BottomNavigationBar:** For smooth navigation between the two test cases.
*   **Asset Management:** Utilizing custom images, colors, and strings (Core constants).
